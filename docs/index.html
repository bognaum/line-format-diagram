HTML-element attributes
data-line-num=""

Line number before a first line of code. It needs to highlight errors of JSON.
Default "0".

data-bd-color=""

Color of all lines.
Default "".

data-style=""

Style to layouted line.
Default "".

style=""

A common style to all structure.

Affects to:

styles of descriptions, like a color, font, etc.

common diagram styles, like display, align background, etc.

JSON template part attributes
"td":

Top description.
"ch":

Child(ren). It may be a string or child template parts.
"bd":

Bottom description.
"style":

Style to expression line.
"tdStyle":

Top description style.
"tbStyle":

Top border style. It may be width and color.
"bdStyle":

Bottom description style.
"bbStyle":

Bottom border style. It may be width and color.
"class":

Additional class to part. It may be used to enable CSS styles.

Description

The top level of JSON template is always list [...].

A "part" is a JSON dict object {...}, which has the one required field, and other optional fields. This required field is "ch":. It may be a string content of layouted expression line, or it may be a list of parent parts.

This is two simple examples.

[ { "td": "2", "ch": "google" }, { "ch": "." }, { "td": "1", "ch": "com" }, { "ch": ".", "style": "color: #f88;", "bd": "a zero level domain \n\nit is not necessary to write this" } ]
[ { "td": "other domains", "ch": [ { "td": "2", "ch": "google" }, { "ch": "." }, { "td": "1", "ch": "com" } ] }, { "ch": ".", "style": "color: #f88;", "bd": "a zero level domain \n\nit is not necessary to write this" } ]
